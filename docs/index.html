Key Highlights

Built different.
At every layer.

Every design decision in Trinity prioritizes deterministic performance, security, and privacy — simultaneously.

01

Zero Remote Dependencies

  • All computation happens on-device
  • No SOCKS5 proxy or remote server required
  • 100% offline operation guaranteed
  • Zero data leaves the user's device
02

Sub-5ms Threat Response

  • Synchronous ML inference on packet thread
  • No coroutine overhead on critical path
  • Thread pinned to CPU core 0 via affinity
  • Hardware-accelerated via XNNPACK/NNAPI
03

Reactive State Management

  • ThreatRepository uses MutableStateFlow
  • Fragments collect with lifecycle awareness
  • UI updates instantly on threat detection
  • Zero memory leaks with proper scoping
04

Singleton Engine Architecture

  • All engines are Kotlin object singletons
  • Loaded once, shared across components
  • Thread-safe init with @Volatile flags
  • Proper shutdown in onDestroy() callbacks
05

AES-256-GCM Model Security

  • .enc files encrypted at rest on disk
  • Decrypted only into protected RAM
  • SHA-256 integrity verified before load
  • Memory explicitly wiped post-inference
06

Crash-Resilient Design

  • nativeStopEngine never closes TUN fd
  • Prevents fdsan double-close on Android 11+
  • Graceful degradation on engine init failure
  • Comprehensive telemetry for all errors
